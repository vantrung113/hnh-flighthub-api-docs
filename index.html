Introduction — 平台简介

What is Flight API Hub?

Flight API Hub is HNH's comprehensive flight booking API service, designed specifically for Online Travel Agencies (OTAs) and travel tech providers. It delivers enterprise-grade capabilities to handle millions of transactions with reliability, speed, and global airline coverage.
Flight API Hub 是 HNH 的综合性航班预订 API 服务，专为在线旅行社 (OTA) 和旅游技术提供商设计。它提供企业级预订功能，以卓越的可靠性、速度及覆盖全球的航司网络，轻松应对海量并发交易。

Booking Management

Authentication — 认证

获取访问令牌 | Obtain Access Token
ℹ️
All API requests require a JWT Bearer token in the Authorization header.
所有API请求都需要在 Authorization 请求头中携带 JWT Bearer 令牌。请联系HNH获取您的 partnerCode、userName 和 password。
POST /api/v1/sign-in 登录 / Sign In ▼

Request Body — 请求体

Field 字段Type 类型Required 必填Description 说明
partnerCode string YES
Partner code assigned by HNHHNH分配的合作伙伴代码，请联系HNH获取
userName string YES
Username provided by HNHHNH提供的用户名
password string YES
Password provided by HNHHNH提供的密码
Request JSON
{
  "partnerCode": "YOUR_PARTNER_CODE",  // 请联系HNH获取 / Contact HNH to obtain
  "userName": "your_username",
  "password": "your_password"
}

Response — 响应

Field 字段Type 类型Description 说明

JWT access token — use in all subsequent requestsJWT访问令牌，后续所有请求都需要携带此令牌

Token for refreshing access without re-login用于刷新访问令牌，避免重新登录

Token expiry datetime令牌过期时间

Expiry duration in seconds令牌有效期（秒）

  "refreshToken": "your_refresh_token_here"

Call this endpoint to get the list of booking systems available to your account. The system code returned here is required in almost every subsequent API call.
调用此接口获取您账户可用的预订系统列表。返回的 system 代码在后续几乎所有接口中都是必填字段。

flights[].endPoint string YES
Destination airport IATA code到达机场的IATA代码

Request Example — SGN→HAN, 1 Adult 示例
{
  "system": "VNA",           // Use system code from /api/v1/system / 从系统列表接口获取
  "adt": 1,               // 1 adult / 1名成人
  "chd": 0,               // 0 children / 0名儿童
  "inf": 0,               // 0 infants / 0名婴儿
  "flights": [
    {
      "leg": 0,             // Outbound / 去程
      "startPoint": "SGN", // Ho Chi Minh City / 胡志明市
      "endPoint": "HAN",   // Hanoi / 河内
      "departDate": "2026-05-15",
      "preferCabin": "Y"  // Economy / 经济舱
    }
  ]
}

Baggage — 行李

查询并添加额外行李额度
POST /api/v1/flight/getbaggages 获取可购行李列表 ▼
Request 请求
{

响应返回 baggages[] 数组，每项包含 prices[].value（编码字符串），添加行李时将此值作为 baggageValue 传入。

Request 请求
{
  "system": "VNA",
  "bookingCode": "ABCDEF",
  "baggages": [
    {
      "quantity": 1,
      "baggageValue": "eyJ..."  // prices[].value from getbaggages / 从行李列表获取
    }
  ]
}

Seat Map — 座位图

查看并选择座位
POST /api/v1/flight/getseatmap 获取航班座位图 ▼
Request 请求
{
  "system": "VNA",
  "flightValue": "eyJ..."  // Flight value from search / 搜索结果中的航班值

Response returns flightSeats[].cabins[].rows[].seats[]. Each seat has available, prices[].value (use as seatValue when booking), and description[] for seat characteristics.
响应返回座位数据，每个座位包含 available（是否可选）、prices[].value（预订座位时用作seatValue）、description[]（座位特性描述）。

POST /api/v1/flight/addseatmap 为旅客选择座位 ▼
Request 请求
{
  "system": "VNA",
  "bookingCode": "ABCDEF",
  "seatMaps": [
    {
      "passengerId": "pax_id_from_retrieve", // From Retrieve response / 从查询预订获取

POST /api/v1/flight/refundticket 退票（两步流程） ▼
ℹ️
2-step process: First call with confirm: false to preview refund amount, then call again with confirm: true to execute.
两步流程：第一次调用时 confirm=false 预览退款金额，确认无误后再以 confirm=true 调用执行退票。
Step 1: Preview 预览退款金额
{
  "system": "VNA",
  "bookingCode": "ABCDEF",
  "ticketNumber": "738-1234567890",
  "confirm": false,   // Dry run — preview only / 仅预览，不执行退票
  "refundAll": false,
  "isEMD": false      // false=flight ticket; true=EMD / false=机票；true=EMD
}
Step 2: Confirm Refund 确认退票
{
  "system": "VNA",
  "bookingCode": "ABCDEF",
  "refundValue": "eyJ...", // From Step 1 response / 来自第一步响应
  "confirm": true
}

// POST /api/v1/flight/select
{ "system": "VNA", "flightValues": ["<flightValue from search>"] }

Step 4: Book 创建预订
// POST /api/v1/flight/book
{
